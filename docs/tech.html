Professional Team

Kazuhiro.M / R&Dスペシャリスト

私は大学卒業後の2006年にJoinし、エンジニアとして広告プロダクトに関しては全ての開発に関わってきました。 Fringe81のエンジニアとして働くなかで心がけてきたことは、新しいことに挑戦し、常に昨日の自分より進化するということです。事業の運営は少数チームで構成され、企画、営業、エンジニア、オペレータ入り乱れて(笑)「あーでもない」「こーでもないと」議論しながら成長させていくスタイルです。事業戦略が共有されるのでビジネスに関する高い視点も磨かれます。その中で、社員は個性的で向上心を持った真面目な人ばかりなので、切磋琢磨され成長を実感できる環境なのが良いところだと感じています。これからもエンジニアの枠を超えた!?進化をつづけます！！！

Keisuke.N / プロジェクトリーダー

私は「問題解決能力」が高ければ高い程、エンジニアとしての価値が高くなると考えています。Fringe81に入社して丸3年の経験を生かし、営業、オペレータ、エンジニアからの相談や質問、常に何かの困難に立ち向かい、解決し、「ありがとう」と言われる事でエンジニアとしてのやりがいを感じています。これからも誰かの役に立ちながら楽しみつつ、Fringe81の今後の成長に足跡を付けて行きたいと思っています。今後は社員エンジニアを増やし、R＆D、新プロダクトの開発、常に生産性を上げる施策を考えアウトプットの質や量は今まで以上にしていきたいと思っています。

Masayuki.A / リードプログラマー

我々が作っているプロダクトは一度作ったらそれで終わりではなく、機能追加や機能改善を行い、常にプロダクトをより良いモノに進化させています。常に進化させるには、品質と開発のスピード感を両立させることが勝負となってきます。そのため、機能/性能要件を満たしたながら、保守性の高いソースコードを表現することも重要となるので、常に1行１行勝負する気持ちでコーディングしています。現在はScalaに可能性を見出だし、Scalaを最大限に活かして戦うことをチームで模索している毎日です！Qiitaでも頑張って連載をしていますので、コメントをお待ちしています！

Javaは約20年の歴史を持ち、小規模なシステムからエンタープライズ領域まで幅広くかつ世界中で利用されている言語で、弊社でも創業当時から一環して、「Java」ないしはその周辺技術でプロダクトを開発/運用してきました。 Javaとは、完全なオブジェクト指向の高等言語で、"Write once, run anywhere" に代表されるメンテナンス性や再利用性、JVMという機種やOS依存しない汎用性、利用者やコミュニティも活発で情報量も多く、学習もし易いことなどが特徴としてあり、人や場所を選ばずチームとして開発するにはとても威力を発揮する言語のひとつだと考えています。

言語の発展と共に、多種多様なコンテナ/フレームワークが生まれ、「AOP」、「DI」、「O/Rマッパー」といった指向、概念、設計思想で開発のし易さや品質向上、そして何よりそういうものを使って開発する「開発の楽しみ」をより体感する事ができました。その一方で、軽量と謳っているそれらも実際の開発現場においては、複雑さを増すケースもあり、本当に便利なのか？これくらいの簡単な機能でもこんな工数がかかるのか？という生産効率という壁に度々ぶつかるようにもなりました。

そこで2012年の年明けに思い切って、「Scala」を採用しました。
もちろんツールレベルでrubyやPHPなどは使用していましたが、実開発に導入するという意味では、大きな決断です。

Scalaは、Javaとの親和性やコードの簡潔性、並列処理、オブジェクト指向とはまた別の概念の関数型へのチャレンジ精神（好奇心）でとにかく始めました。 まだまだ未熟な部分も多く、一つ一つ地道な階段を登っていますが、何でそうなっているのかとか、どうしてそう動いているのかとか、しっかりと時間をかけながら、徐々に熟成の域まで持っていきたいと思い、以下のような取り組みを積極的におこなっております。
